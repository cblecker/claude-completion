# Claude Completion Project Conventions

## Architecture

- **Data-driven**: Parser extracts CLI help into JSON, generator produces bash completion from JSON
- `completions/claude.bash` is generated — edit `scripts/generate_completion.py` or `data/claude-completion.json`, not the bash file directly
- `data/claude-completion.json` is generated by the parser — edit `scripts/parse_help.py`, not the JSON directly

## Scripts

- Python 3 stdlib only (no pip dependencies)
- All script invocations go through `make` targets

## Make Targets

- `make parse` — run parser against live `claude --help`, produces JSON + VERSION
- `make generate` — read JSON, produce bash completion script
- `make update` — parse + generate (used by automation)
- `make validate` — check JSON validity + bash syntax
- `make test` — validate + run functional tests
- `make verify-generation` — ensure generated script matches data (no manual edits)
